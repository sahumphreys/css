Introduction

For the new OCR GCSE Computer Science (J277) a reference language has been introduced: the OCR Reference Language (OCL). The goal is to ensure some consistency across the questions being asked but it can also be used for answering programming questions too.

Thus, a question might include the following code fragment using OCL to carry out validation on data input by the user:

valid = false while(valid == false) studentName = input("Enter the student name") if (studentName.length > 0) AND (studentName.length < 30) valid = true else print("Student name is not valid") endif endwhile

The assignment of variables, dealing with input and output looks similar to Python: but the loop and if statement are different:

  • A colon (':') is not used at the end of these lines
  • The if statement has a closing endif
  • The while statement has a closing endwhile

This document covers the key components of the reference language and compares the language to similar constructs and statements used in Python.

Comments

Comments are used to explain why a particular sequence of instructions, or algorithm, is being used. They act as "notes to self" and play an important job of keeping your code maintainable.

There is a balance to be kept here. Too many comments are unnecessary noise, and too few can make your code harder to understand for others. We can cut down on the number of comments by using sensible naming conventions e.g.

// initialising the score to zero x = 0

We could avoid using a comment by changing the name of the variable e.g.

score = 0

Python uses '#' for comments. Here, two forward slashes, '//' are used.

Operators

The standard set of operators are used in ORL:

Arithmetic

For example:

n = 4 + 5 // Addition
n = 12 - 9 // Subtraction
n = 3 * 7 // Multiplication
n = 12 / 4 // Division
n = 7 DIV 2 // Quotient
n = 7 MOD 2 // Modulus

NB. Integer division and modulus are different to Python

Comparison

For example:

age == 18 // Equality
score != 0 // Not equal to
total < 0 // Less than
total > 0 // greater than
sum <= 50 // less than or equal to
sum >= 50 // greater than or equal to

Logical

For example:

age >= 11 AND age <= 18 // AND
score < 10 OR score > 5 // OR
NOT day == "Monday" // NOT

These are all the same as used in Python

Do not forget:

  • Do not confuse the equality operator ("==") with the assignment operator ("!=")
  • Always check if the "=" needs to be included when using greater than or less than comparison operators
